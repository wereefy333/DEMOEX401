Универсальный помощник по модулям

Материал собран для подготовки: как быстро построить ER-диаграмму, создать таблицы в pgAdmin, импортировать данные и написать запрос на стоимость заказа. Это не готовое списывание, а рабочий шаблон мышления под типовой вариант.

Что повторяется почти во всех вариантах

  • Есть контрагенты: заказчики, поставщики, изготовители.
  • Есть продукция и материалы, у которых обычно есть цена и единица измерения.
  • Есть спецификация продукции: какие материалы и в каком количестве нужны на 1 единицу продукции.
  • Есть заказ покупателя и строки заказа: какая продукция, сколько штук, по какой цене.
  • Есть задача на итоговую стоимость заказа через количество продукции и себестоимость материалов.

Быстрый план на экзамене

  1. Сначала выпиши сущности из файлов: кто, что продаёт, что производится, из чего состоит.
  2. Отдельно вынеси справочники: контрагенты, продукция, материалы, единицы измерения.
  3. Для связей многие-ко-многим сразу делай промежуточные таблицы.
  4. После ER сразу создавай таблицы с первичными ключами, внешними ключами и NOT NULL.
  5. Только потом импортируй JSON и проверяй запросы.

Универсальная схема сущностей для модулей 1-3

unit
  id, name

counterparties
  id, external_code, name, inn, address, phone, is_supplier, is_customer

products
  id, code, name, unit_id, sale_price
  FK: unit_id -> unit.id

materials
  id, code, name, unit_id, price
  FK: unit_id -> unit.id

specifications
  id, name, product_id, output_quantity
  FK: product_id -> products.id

specification_items
  id, specification_id, material_id, material_quantity
  FK: specification_id -> specifications.id
  FK: material_id -> materials.id

customer_orders
  id, order_number, order_date, seller_id, customer_id
  FK: seller_id -> counterparties.id
  FK: customer_id -> counterparties.id

customer_order_items
  id, order_id, product_id, quantity, unit_price
  FK: order_id -> customer_orders.id
  FK: product_id -> products.id

users
  id, email, passwordhash, role, failed_attempts, blocked_until, is_blocked

Логика простая: одна продукция может иметь много материалов, поэтому связь products ↔ materials делается через specifications и specification_items.

Как быстро собрать ER-диаграмму в pgAdmin

  1. Создать таблицу: Tables -> Create -> Table...
  2. Колонки: во вкладке Columns добавь поле, тип, Primary Key, Not Null.
  3. Связи: во вкладке Constraints -> Foreign Key создай внешний ключ.
  4. ERD: нажми правой кнопкой по базе данных или схеме и выбери ERD For Database.
  5. Экспорт: в окне ERD скачай схему как изображение или PDF, если это нужно для отчёта.

Для хорошей 3НФ не храни в одной таблице и заказ, и список материалов, и контрагента одновременно. Справочники отдельно, документы отдельно, строки документов отдельно.

Какие таблицы чаще всего нужны именно в этом КОД

  • counterparties: импорт из Заказчики.json.
  • unit: единицы измерения, например шт, кг, л.
  • products: продукция из заказа, производства и прайса.
  • materials: сырьё из спецификации и производства.
  • specifications и specification_items: состав продукции.
  • customer_orders и customer_order_items: заказ покупателя.
  • users: для модуля авторизации.

Шаблон DDL для базы данных

CREATE TABLE counterparties (
    id SERIAL PRIMARY KEY,
    external_code VARCHAR(20) UNIQUE,
    name VARCHAR(255) NOT NULL,
    inn VARCHAR(20),
    address TEXT,
    phone VARCHAR(30),
    is_supplier BOOLEAN DEFAULT FALSE,
    is_customer BOOLEAN DEFAULT FALSE
);

CREATE TABLE unit (
    id SERIAL PRIMARY KEY,
    name VARCHAR(20) NOT NULL UNIQUE
);

CREATE TABLE products (
    id SERIAL PRIMARY KEY,
    code VARCHAR(20) UNIQUE,
    name VARCHAR(255) NOT NULL UNIQUE,
    unit_id INTEGER NOT NULL REFERENCES unit(id),
    sale_price NUMERIC(12, 2)
);

CREATE TABLE materials (
    id SERIAL PRIMARY KEY,
    code VARCHAR(20) UNIQUE,
    name VARCHAR(255) NOT NULL UNIQUE,
    unit_id INTEGER NOT NULL REFERENCES unit(id),
    price NUMERIC(12, 2) NOT NULL
);

CREATE TABLE specifications (
    id SERIAL PRIMARY KEY,
    name VARCHAR(255) NOT NULL,
    product_id INTEGER NOT NULL REFERENCES products(id) ON DELETE CASCADE,
    output_quantity NUMERIC(12, 3) NOT NULL DEFAULT 1
);

CREATE TABLE specification_items (
    id SERIAL PRIMARY KEY,
    specification_id INTEGER NOT NULL REFERENCES specifications(id) ON DELETE CASCADE,
    material_id INTEGER NOT NULL REFERENCES materials(id),
    material_quantity NUMERIC(12, 3) NOT NULL,
    UNIQUE (specification_id, material_id)
);

CREATE TABLE customer_orders (
    id SERIAL PRIMARY KEY,
    order_number VARCHAR(30) NOT NULL UNIQUE,
    order_date DATE NOT NULL,
    seller_id INTEGER REFERENCES counterparties(id),
    customer_id INTEGER NOT NULL REFERENCES counterparties(id)
);

CREATE TABLE customer_order_items (
    id SERIAL PRIMARY KEY,
    order_id INTEGER NOT NULL REFERENCES customer_orders(id) ON DELETE CASCADE,
    product_id INTEGER NOT NULL REFERENCES products(id),
    quantity NUMERIC(12, 3) NOT NULL,
    unit_price NUMERIC(12, 2),
    UNIQUE (order_id, product_id)
);

Шаблон запроса для модуля 3

SELECT
    customer_orders.order_number,
    SUM(
        customer_order_items.quantity
        * specification_items.material_quantity
        * materials.price
    ) AS order_material_cost
FROM customer_orders
JOIN customer_order_items
    ON customer_order_items.order_id = customer_orders.id
JOIN products
    ON products.id = customer_order_items.product_id
JOIN specifications
    ON specifications.product_id = products.id
JOIN specification_items
    ON specification_items.specification_id = specifications.id
JOIN materials
    ON materials.id = specification_items.material_id
GROUP BY customer_orders.order_number
ORDER BY customer_orders.order_number;

Это самый простой шаблон под типовой модуль 3. Если в варианте поменяют названия, тебе обычно нужно заменить только имена таблиц, полей заказа, количества, нормы расхода и цены материала.

Как читается этот запрос построчно

  1. SELECT начинает выборку данных, которые ты хочешь увидеть в результате.
  2. customer_orders.order_number выводит номер заказа, чтобы было понятно, для какого заказа считается сумма.
  3. SUM(...) суммирует стоимость материалов по всем строкам и всем материалам внутри заказа.
  4. customer_order_items.quantity это количество продукции в заказе.
  5. specification_items.material_quantity это норма расхода материала на одну единицу продукции.
  6. materials.price это цена одного материала.
  7. Когда ты умножаешь количество продукции на норму расхода и на цену материала, получаешь стоимость материала для этой позиции.
  8. AS order_material_cost задаёт удобное имя итоговому столбцу.
  9. FROM customer_orders показывает, с какой основной таблицы начинается построение запроса.
  10. JOIN customer_order_items ON customer_order_items.order_id = customer_orders.id присоединяет строки заказа к самому заказу.
  11. JOIN products ON products.id = customer_order_items.product_id присоединяет продукцию, которая указана в строке заказа.
  12. JOIN specifications ON specifications.product_id = products.id находит спецификацию для этой продукции.
  13. JOIN specification_items ON specification_items.specification_id = specifications.id получает материалы, входящие в спецификацию.
  14. JOIN materials ON materials.id = specification_items.material_id получает цену и название материала.
  15. GROUP BY customer_orders.order_number собирает все строки одного заказа в одну итоговую строку.
  16. ORDER BY customer_orders.order_number просто сортирует результат по номеру заказа.

Как применить запрос на своём варианте

  1. Найди таблицу заказа. Это аналог customer_orders.
  2. Найди таблицу строк заказа. Это аналог customer_order_items.
  3. Найди таблицу продукции. Это аналог products.
  4. Найди таблицу спецификации или состава. Это аналог specifications и specification_items.
  5. Найди таблицу материалов. Это аналог materials.
  6. Проверь 4 ключевых поля: номер заказа, количество продукции, норма расхода материала, цена материала.
  7. После этого просто замени названия таблиц и полей в шаблоне, не меняя саму логику соединений.
Что обычно заменяется:
customer_orders                 -> ваша таблица заказов
customer_order_items            -> ваша таблица строк заказа
customer_order_items.quantity   -> ваше поле количества
specification_items             -> ваша таблица состава
specification_items.material_quantity -> ваше поле нормы расхода
materials                       -> ваша таблица материалов
materials.price                 -> ваше поле цены

name       -> name
inn        -> inn
addres     -> address

Да, в JSON поле называется addres без второй буквы s — это нормально, просто учитывай это при импорте.

Что держать в голове для модуля 4

  • Нужны роли Администратор и Пользователь.
  • После 3 ошибок логина или капчи запись блокируется.
  • Админ должен видеть список пользователей, добавление и снятие блокировки.
  • Для хранения пароля лучше использовать хэш, а не открытый текст.

Частые ошибки, за которые режут баллы

  • Нет промежуточной таблицы для состава спецификации или состава заказа.
  • Связи есть на диаграмме, но не созданы внешние ключи в базе.
  • Сущности смешаны: материалы и продукция лежат в одной таблице без чёткой причины.
  • NULL разрешён там, где по смыслу значение обязательно.
  • Запрос считает только цену продажи и забывает материалы, или наоборот.
  • Имена таблиц и полей хаотичные и не согласованы между ER и БД.

import tkinter as tk
from tkinter import messagebox, ttk
import database
from datetime import datetime, timedelta

try:
    from PIL import Image, ImageTk
    PIL_AVAILABLE = True
except ImportError:
    PIL_AVAILABLE = False
    print("Библиотека Pillow не установлена. CAPTCHA будет работать в упрощённом режиме.")
    print("Установите: pip install Pillow")

import os

class AuthApp:
    def __init__(self, root):
        self.root = root
        self.root.title("Система авторизации - CompanyName")
        self.root.minsize(500, 500)
        self.root.columnconfigure(0, weight=1)
        
        self.captcha_images = [
            "images/1.png", "images/2.png", 
            "images/3.png", "images/4.png"
        ]
        
        self.correct_order = [0, 1, 2, 3]
        self.current_order = [None, None, None, None]
        
        self.auth_frame = self.create_auth_frame()
        self.captcha_frame = None
        self.admin_frame = None
        
        self.show_auth_screen()

    def create_auth_frame(self):
        frame = ttk.Frame(self.root, padding=20)
        frame.grid(row=0, column=0, sticky="nsew")
        
        ttk.Label(frame, text="Авторизация", font=("Arial", 16)).grid(
            row=0, column=0, columnspan=2, pady=20)
        
        ttk.Label(frame, text="Email:").grid(row=1, column=0, sticky="e", pady=5)
        self.email_var = tk.StringVar()
        ttk.Entry(frame, textvariable=self.email_var, width=30).grid(
            row=1, column=1, padx=10, pady=5, sticky="ew")
        
        ttk.Label(frame, text="Пароль:").grid(row=2, column=0, sticky="e", pady=5)
        self.password_var = tk.StringVar()
        ttk.Entry(frame, textvariable=self.password_var, show="*", width=30).grid(
            row=2, column=1, padx=10, pady=5, sticky="ew")
        
        ttk.Button(frame, text="Войти", command=self.login).grid(
            row=3, column=0, columnspan=2, pady=20)
        
        return frame

    def show_auth_screen(self):
        if self.admin_frame:
            self.admin_frame.destroy()
            self.admin_frame = None
        if self.captcha_frame:
            self.captcha_frame.destroy()
            self.captcha_frame = None
        if self.auth_frame:
            self.auth_frame.tkraise()

    def login(self):
        email = self.email_var.get().strip()
        password = self.password_var.get().strip()
        
        if not email or not password:
            messagebox.showerror("Ошибка", "Заполните все поля!")
            return
        
        user, error = database.authenticate_user(email, password)
        
        if error == "blocked":
            messagebox.showerror("Ошибка", "Вы заблокированы. Обратитесь к администратору")
        elif error:
            messagebox.showerror("Ошибка", error)
        elif user:
            self.current_user_for_captcha = user
            self.show_captcha_screen()

    def show_captcha_screen(self):
        self.auth_frame.grid_forget()
        
        self.captcha_frame = ttk.Frame(self.root, padding=20)
        self.captcha_frame.grid(row=0, column=0, sticky="nsew")
        
        ttk.Label(self.captcha_frame, text="Соберите изображение", 
                 font=("Arial", 14)).grid(row=0, column=0, columnspan=2, pady=10)
        
        self.photo_images = []
        for i in range(4):
            if PIL_AVAILABLE and os.path.exists(self.captcha_images[i]):
                try:
                    img = Image.open(self.captcha_images[i])
                    img = img.resize((150, 100), Image.Resampling.LANCZOS)
                    photo = ImageTk.PhotoImage(img)
                    self.photo_images.append(photo)
                except Exception as e:
                    print(f"Ошибка загрузки {self.captcha_images[i]}: {e}")
                    self._create_placeholder(i)
            else:
                self._create_placeholder(i)
        
        button_frame = ttk.Frame(self.captcha_frame)
        button_frame.grid(row=1, column=0, columnspan=2, pady=10)
        
        self.captcha_buttons = []
        for i in range(4):
            btn = tk.Button(button_frame, image=self.photo_images[i],
                           command=lambda idx=i: self.move_captcha_piece(idx))
            btn.grid(row=0, column=i, padx=5)
            self.captcha_buttons.append(btn)
        
        self.assembly_frame = ttk.Frame(self.captcha_frame, relief="solid", borderwidth=2)
        self.assembly_frame.grid(row=2, column=0, columnspan=2, pady=10)
        
        self.assembly_labels = []
        positions = ["Верх-лево", "Верх-право", "Низ-лево", "Низ-право"]
        for i in range(4):
            row, col = divmod(i, 2)
            frame = ttk.Frame(self.assembly_frame, width=160, height=110, relief="ridge")
            frame.grid(row=row, column=col, padx=5, pady=5)
            frame.pack_propagate(False)
            
            label = ttk.Label(frame, text=positions[i], anchor="center")
            label.pack(expand=True)
            self.assembly_labels.append(label)
        
        btn_frame = ttk.Frame(self.captcha_frame)
        btn_frame.grid(row=3, column=0, columnspan=2, pady=15)
        ttk.Button(btn_frame, text="Проверить", command=self.check_captcha).pack(side="left", padx=5)
        ttk.Button(btn_frame, text="Назад", command=self.show_auth_screen).pack(side="left", padx=5)
        
        self.current_order = [None, None, None, None]

    def _create_placeholder(self, index):
        if PIL_AVAILABLE:
            from PIL import Image, ImageDraw
            img = Image.new('RGB', (150, 100), color='lightgray')
            draw = ImageDraw.Draw(img)
            draw.text((60, 40), f"ФРАГМЕНТ\n{index+1}", fill='black')
            photo = ImageTk.PhotoImage(img)
        else:
            photo = None
        self.photo_images.append(photo)

    def move_captcha_piece(self, piece_index):
        for i, label in enumerate(self.assembly_labels):
            if not hasattr(label, '_piece_index'):
                if self.photo_images[piece_index]:
                    label.config(image=self.photo_images[piece_index], text="")
                else:
                    label.config(text=f"ФРАГМЕНТ {piece_index+1}")
                label._piece_index = piece_index
                self.current_order[i] = piece_index
                return
        messagebox.showinfo("Информация", "Все позиции заполнены!")

    def check_captcha(self):
        if None in self.current_order:
            messagebox.showerror("Ошибка", "Заполните все позиции!")
            return
        
        if self.current_order == self.correct_order:
            messagebox.showinfo("Успешно", "CAPTCHA пройдена!")
            user = self.current_user_for_captcha
            if user['role'] == 'Администратор':
                self.show_admin_screen(user)
            else:
                self.root.destroy()
                self.show_user_screen(user)
        else:
            email = self.current_user_for_captcha['email']
            
            conn = database.get_db_connection()
            cursor = conn.cursor()
            cursor.execute("SELECT failed_attempts FROM users WHERE email = %s", (email,))
            row = cursor.fetchone()
            failed_attempts = row[0] if row else 0
            
            new_attempts = failed_attempts + 1
            if new_attempts >= 3:
                blocked_time = datetime.now() + timedelta(hours=24)
                cursor.execute("""
                    UPDATE users 
                    SET failed_attempts = %s, blocked_until = %s 
                    WHERE email = %s

    try:
        conn = get_db_connection()
        cursor = conn.cursor()
        
        cursor.execute("""
            SELECT email, role, blocked_until, failed_attempts, is_blocked

            WHERE email = %s

    global ssh_tunnel

    if ssh_tunnel and ssh_tunnel.is_active:
        return ssh_tunnel

    ssh_tunnel = SSHTunnelForwarder(
        (SSH_CONFIG["ssh_host"], SSH_CONFIG["ssh_port"]),
        ssh_username=SSH_CONFIG["ssh_username"],
        ssh_pkey=SSH_CONFIG["ssh_private_key"],
        ssh_private_key_password=SSH_CONFIG["ssh_private_key_password"],
        remote_bind_address=(REMOTE_DB_CONFIG["host"], REMOTE_DB_CONFIG["port"])
    )
    ssh_tunnel.start()
    print(f"SSH-туннель запущен: 127.0.0.1:{ssh_tunnel.local_bind_port}")
    return ssh_tunnel

    global ssh_tunnel
